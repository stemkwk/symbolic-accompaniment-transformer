## 🔍 상세 해결 내용

### 1. 추론 시점의 인과성 누수 버그
* **위치**: `src/jam_transformer/model.py`
* **해결**: 추론 시 멜로디 조건(`forced` 토큰) 주입 블록(`T > 1`)에서 비인과적(Non-causal) 어텐션이 적용되던 현상을 수정하였습니다. `kv_cache`가 있고 입력 시퀀스 길이가 1보다 클 때, 미래 토큰을 참조하지 못하도록 2D 쿼리-키 마스크(`attn_mask`)를 직접 생성하여 주입하도록 개선했습니다.

### 2. Top-P (Nucleus) 샘플링 경계선 누락 버그
* **위치**: `src/jam_transformer/model.py`
* **해결**: 누적 확률 분포에서 `top_p` 임계값을 초과하는 첫 번째 경계선 토큰이 후보군에서 배제되던 문제를 수정했습니다. 누적 합(`cum`) 비교 범위를 한 칸 시프트(`cum[..., :-1] < top_p`)하여 원래 Nucleus 샘플링 규격에 맞게 동작하도록 교정했습니다.

### 3. Humanizer의 시간 단위 변환 오작동 버그
* **위치**: `src/jam_transformer/midi_io.py`
* **해결**: `tempo` 변수의 단위가 마이크로초가 아닌 BPM(Beats Per Minute)인 점을 반영하여, `ms_per_tick = (60000.0 / tempo) / tpb` 수식으로 수정했습니다. 이로 인해 타이밍 틱 오차 변환 오작동 및 박자 붕괴 문제가 완전히 해결되었습니다.

### 4. LR 스케줄러 과소 스텝 버그
* **위치**: `scripts/train.py`
* **해결**: Gradient Accumulation(`accumulate_grad_batches > 1`) 활성화 시 실제 가중치 업데이트 주기와 스케줄러 스텝의 불일치로 러닝 레이트가 끝까지 감쇠하지 않던 버그를 수정했습니다. 에폭당 유효 스텝 수(`steps_per_epoch // accum_batches`)를 기준으로 `total_steps`를 계산하도록 보완했습니다.

### 5. Legay 보폭 분할의 인덱스 이중 활용 버그
* **위치**: `scripts/train.py`
* **해결**: 학습용 데이터셋(`full`)과 검증용 데이터셋(`full_val`)의 청크 개수 차이로 인해 인덱스 에러가 발생하던 문제를 해결했습니다. `val_idx` 리스트를 생성할 때 `full_val._chunks`를 기준으로 인덱스를 정확하게 탐색하도록 교정했습니다.

### 6. Gradio 오디오 루프 스테이션 내 데이터 타입 불일치 및 클리핑 버그
* **위치**: `app.py`
* **해결**: 마이크 녹음(int16)과 FluidSynth 반주(float32)의 스케일 차이로 오디오가 묻히거나 노이즈로 뭉개지는 현상을 방지하기 위해, 두 오디오 데이터를 모두 `float32` [-1.0, 1.0] 범위로 정규화한 뒤 믹싱하고, 최종 출력 시 `int16` 범위로 복원하여 클리핑 없이 선명하게 합성하도록 개선했습니다.

### 7. Gradio 시연 시 오디오 합성 우선순위 교정
* **위치**: `app.py`
* **해결**: MIDI 파일과 사용자의 오디오/마이크 입력이 동시에 존재하는 경우, 반주 생성 자체는 오정보가 없는 깨끗한 MIDI 파일을 기반으로 동작시켜 완성도를 높이고, 최종 WAV 합성 시에는 사용자의 원본 녹음 오디오를 최우선으로 사용하여 AI 반주와 자연스럽게 믹싱되도록 제어 흐름을 수정했습니다.
