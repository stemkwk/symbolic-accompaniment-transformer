# JAM Transformer 시연 및 파이프라인 고도화 분석 리포트

## 1. 추론 (Inference) 아키텍처 및 웹 데모(UI) 평가

### 1.1. 추론 아키텍처 (model.py)
* **KV-Cache 최적화**: O(N²) 복잡도를 O(N)으로 낮추는 KV-Cache가 완벽히 적용되어 있어 실시간 생성에 유리합니다.
* **CFG(Classifier-Free Guidance) 병렬화**: 조건부/무조건부 프롬프트를 한 번의 배치(Batch)로 동시에 연산하여 속도와 퀄리티를 모두 잡았습니다.
* **유연한 샘플러**: 음악의 기계적인 단조로움을 피하기 위한 `Top-k`, `Top-p(Nucleus Sampling)`, `Temperature` 스케일링이 정석적으로 구현되어 있습니다.

### 1.2. 웹 데모 UI (app.py)
* **루프 스테이션 (Loop Station) 모드**: 사용자가 연주하는 15~30초 동안 AI가 백그라운드에서 반주를 생성하여 실시간으로 레이어를 덧입히는 창의적이고 완성도 높은 라이브 잼(Jam) 워크플로우를 제공합니다.
* **강력한 오디오 파이프라인**: 
    * `librosa`를 활용한 자동 BPM 추적
    * `pedalboard` DSP 이펙터(리버브, 컴프레서 등)를 적용해 FluidSynth의 투박한 사운드를 상용 가상악기 수준으로 끌어올림
    * 오디오 포맷별(8-bit, 16-bit, int32 등) Clipping 방지 정규화 완비

## 2. 시연을 위한 하드웨어 입력 장비 제언

**Audio-to-MIDI 변환기의 정확도**는 최종 AI 반주의 퀄리티를 결정짓는 핵심 요소입니다. 입력 소스가 깨끗할수록 결과물이 완벽해집니다.

* **최상위 추천 (안정성 100%)**: **마스터 키보드** (가상악기 경유) 또는 **일렉 기타** (오디오 인터페이스 다이렉트 연결)
    * 이유: 피치가 뚜렷하고 어택(Onset)이 선명하여 `audio_to_midi`가 음표를 100% 정확하게 인식합니다.
* **하위 추천 (위험도 높음)**: **노트북 내장 마이크를 통한 보컬(목소리) 입력**
    * 이유: 방의 울림, 팬 소음, 그리고 보컬 특유의 비브라토(떨림)와 포르타멘토(미끄러짐)로 인해 기계가 원치 않는 **유령 음표(Ghost notes)**를 무더기로 생성하여 반주가 붕괴될 위험이 큽니다.

## 3. "보컬 음성 변환 (Voice Conversion)" 병렬 파이프라인 고도화

다른 프로젝트인 '보컬 음성 변환 ML'과 본 'JAM Transformer'를 **병렬(Parallel)로 처리하여 최종 믹싱**하는 시연 시나리오를 위한 기술적 고도화 방안입니다.

### 3.1. Audio-to-MIDI (단일 장애점) 튜닝의 필요성
AI 모델은 실제 오디오(WAV)를 듣지 못하고, 오직 변환된 악보(MIDI)만 보고 반주를 생성합니다. 즉 **보컬 오디오의 MIDI 변환 퀄리티가 병렬 파이프라인 전체의 성패를 좌우합니다.** 보컬의 잔떨림이 음표로 치환되는 현상을 방지하기 위해 `configs/config.yaml`의 파라미터를 보컬에 맞게 튜닝해야 합니다.

### 3.2. 보컬 특화 파라미터 최적화 권장 수치
1. **`min_note_length_ms` (최소 음표 길이)**
    * **기존**: `58.0` 
    * **변경 권장**: `150.0` ~ `200.0`
    * **효과**: 보컬이 음을 끌 때 발생하는 비브라토나 미세한 음정 흔들림을 무시하고 굵직한 핵심 멜로디만 토큰화합니다.
2. **`onset_threshold` (음표 시작점 민감도)**
    * **기존**: `0.5`
    * **변경 권장**: `0.6` ~ `0.7`
    * **효과**: 숨소리(Breath), 립스맥(Lip smack) 등 불필요한 잡음을 음표로 잘못 인식하는 빈도를 줄여줍니다.
3. **노이즈 제거 (`denoise`) 유지**
    * 변환된 오디오가 아닌 마이크 원본 오디오가 Jam Transformer의 입력으로 들어가므로, 노이즈 제거 기능(`denoise=True`)을 활성화하여 주변 화이트 노이즈를 억제해야 피치 추적에 유리합니다.

### 3.3. 최종 믹싱 동기화 (Sync) 체크
병렬 파이프라인의 핵심은 두 모델 출력값의 **정확한 박자 일치**입니다. 음성 변환 모델이 오디오를 출력할 때 앞뒤로 무음(Padding)을 추가하거나 딜레이를 유발하지 않는지 사전에 점검하여, 최종 합성 시 반주와 보컬이 엇박자가 나지 않도록 유의해야 합니다. (가능하다면 입력 전 오토튠 등 가벼운 피치 보정을 거치면 인식률이 극대화됩니다.)
