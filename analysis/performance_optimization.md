## 🔍 완료된 최적화 세부 내용

### 1. 데이터 증강 시의 CPU 병목 현상 (학습 중 GPU Starvation 유발)
* **문제점**: 이전 버전에서는 전조 증강(Pitch Transposition) 시 토큰 시퀀스 전체를 순회하는 순수 Python `for` 루프가 동작하여 CPU 단의 데이터 로딩 연산 속도가 심각하게 지연되고 GPU 활용률이 하락하는 병목이 있었습니다.
* **해결 내용**: 
  - 복잡한 순차 루프를 제거하고 PyTorch의 텐서 병렬 연산 및 슬라이싱 기법을 적용하여 $O(N)$ 연산을 $O(1)$ 벡터화 연산으로 완벽히 변환하였습니다.
  - 이를 통해 배치 생성 시 CPU 병목을 소멸시켜 GPU 가동률(Utility)을 대폭 향상시켰습니다.

## 💡 미반영 제안 (향후 고도화 과제)

### 1. Dynamic KV 캐시 병합으로 인한 메모리 단편화 및 할당 오버헤드
* **제안 배경**: `CausalSelfAttention`에서 매 디코딩 단계마다 캐시를 병합하기 위해 `torch.cat`을 호출하며, 이로 인해 GPU 메모리 할당 및 복사가 반복되어 메모리 단편화와 오버헤드가 누적되고 있습니다.
* **구현 방안**: 추론 시작 시 최대 출력 길이(`max_seq_len`) 크기만큼의 static KV 캐시 버퍼를 GPU 메모리에 미리 할당(Pre-allocation)해두고, 쓰기 포인터(Write Pointer)를 통해 값을 덮어쓰는 구조로 개선하면 메모리 복사 오버헤드를 완전히 소멸시킬 수 있습니다.

### 2. 슬라이딩 윈도우(Sliding Window) 재계산 오버헤드
* **제안 배경**: 생성 길이가 `ctx_window`를 초과할 경우 기존 KV 캐시를 버리고, 헤더 토큰과 최근 `ctx_keep`개의 토큰을 이어 붙여 모델에 처음부터 다시 통과시켜 새로운 KV 캐시를 빌드합니다. 이로 인해 컨텍스트 슬라이딩 시점에 연산 레이턴시가 튀는 현상이 있습니다.
* **구현 방안**: RoPE(회전식 위치 임베딩)의 상대 위치 보존 성질을 활용하여, 기존 KV 캐시 텐서에서 앞부분을 잘라내고(Slice) 뒤로 이어붙이는 인플레이스(In-place) 캐시 슬라이딩 방식을 도입함으로써 재연산 비용을 $O(T_{\text{window}})$에서 $O(1)$로 축소하는 고도화가 가능합니다.
